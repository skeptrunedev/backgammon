# Backgammon vs GNU Backgammon

A fully client-side backgammon PWA. Play matches against the GNU Backgammon
neural-net engine (the same engine used by serious players for analysis),
running entirely in your browser via WebAssembly — no backend, works offline.

## Features

- **World-class opponent** — gnubg 2-ply neural-net checker play and cube
  decisions (strength configurable: 0/1/2-ply)
- **Full match play** — doubling cube, Crawford rule, gammons/backgammons,
  resignations, any match length
- **Automatic analysis of every decision you make** — while you think, the
  engine ranks every legal move (and every cube action). At match end you have
  a complete record: every mistake, its equity cost, and the win-probability
  difference vs the best play
- **Match history** — every match is stored locally (IndexedDB) with per-decision
  analysis: blunders (≥0.08 eq), errors (≥0.04), dubious plays (≥0.02)
- **.mat export** — download any match in the standard Jellyfish `.mat` format
  (generated by gnubg itself) for import into XG, gnubg desktop, etc.
- **AI explanations** — for any suboptimal play, ask Claude to explain *why*
  the engine's choice is better, grounded in the engine's ranked evaluations
  (bring your own Anthropic API key; stored only in localStorage)

## Setup

```bash
npm install
npm run fetch-engine   # downloads the gnubg WASM build (see licensing below)
npm run dev
```

Other commands:

```bash
npm run build    # typecheck + production build (PWA)
npm test         # rules-engine unit tests
npm run smoke    # drives the real WASM engine through a full 3-point match in Node
```

## Architecture

```
src/engine/   gnubg WASM in a Web Worker; promise RPC; parsers for the FIBS
              board format, hint rankings, and cube analysis output
src/game/     TypeScript rules engine (legal-move generation for the UI),
              session state machine, match records + IndexedDB storage
src/ui/       React + Tailwind + shadcn; SVG board with click-to-move
src/ai/       Claude API explanation prompts (position + engine evals)
```

The gnubg engine is authoritative for everything: it rolls the dice, validates
moves, plays the opponent, evaluates equities, and writes the `.mat` file. The
TypeScript rules engine exists only to make the board interactive (highlighting
legal moves) — every move you commit is executed by gnubg itself.

Analysis happens *during* play: when you roll, the engine's `hint` ranking for
your position is computed in the background while you think, so the post-match
mistake report is ready the moment the match ends. Checker-play equity losses
use gnubg's move filters (top candidates at full ply, the rest at 0-ply), the
same trade-off gnubg's own analysis uses.
